## What was adapted into MM in this revision

FT8 RX now always uses an MSHV-style deep/multipass loop:

1. Search and decode candidates.
2. Deduplicate confirmed CRC-valid decodes.
3. Reconstruct each decoded message's 79 FT8 tones using the MSHV generator
   tables.
4. Subtract the reconstructed codeword from a fresh working copy of the slot.
5. Search/decode again to expose weaker or overlapped signals.
6. Repeat for a third pass.

This replaces the previous state where multipass existed only behind the old
`enhanced` pseudo-flavour.  Since MM now has one MSHV pipeline, deep decode is
part of that pipeline.

## TX

FT8 TX was already generated by MSHV `GenFt8`.  FT4 TX is now generated by MSHV
`GenFt4` as well.  The local hand-built FT4 waveform was removed from the active
TX path.

## What remains different from full MSHV

MM still uses its compact Goertzel/Qt worker receiver rather than importing the
entire MSHV decoder object.  Therefore this is not yet a bit-identical MSHV
receiver.  The next exactness step would be to wrap or port the full
`DecoderFt8::sync8()`, `ft8b()` and `subtractft8()` stack including the FFT
matrix `s_`, `sbase`, AP state and the exact K_SUB matched-filter subtractor.
