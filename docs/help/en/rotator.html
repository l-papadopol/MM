Rotator

The rotator is independent from radio CAT. Each Rotator 1/2/3 profile uses a separate Hamlib rotator endpoint and can be assigned to different bands.

Mandatory station identity

Before moving the rotator or transmitting, configure a valid My Call and My Locator in Settings → User/QTH. If they are missing, MM blocks TX, PTT, Tune, calibration and rotator pointing.

Band table

At the bottom of each rotator profile there is a band table from 40 m to 10 GHz with use checkbox, AZ/EL search span in degrees and Auto peak per band. The old band string is no longer the user interface.

Signal peak search

The algorithm list leaves selectable only algorithms compatible with the available axes. Incompatible algorithms remain visible but struck through. Search must stay inside the configured span and must not move the rotator during TX.

Manual target pointing

In the Rotator tab you can type a Maidenhead locator, country/DXCC or prefix from cty.csv. MM computes bearing and distance from your QTH and points the active rotator.

MOV/RDY lamps

Bright red MOV means rotator moving; bright green RDY means configured, connected and stopped. If no rotator is configured, both lamps are greyed.

Azimuth geometry

Settings → Rotator lets each profile describe the real mechanical scale: 360° stop at North, 360° stop at South, Yaesu 450° overlap (N-E-S-W-N-E), or Custom. If end-stop auto-reverse is enabled, MM retries the other valid mechanical path when the rotor is commanded but does not move appreciably within the configured timeout.

Calibration

When auto-calibration finishes, the measured ms/degree values and calibration label update immediately in the Settings dialog.

Navball display

The Rotator side tab uses a Navball display: the yellow fixed reticle is the current antenna pointing, the green TG marker is the active target, and OVERLAP ACTIVE appears when the mechanical azimuth is above 360°.
