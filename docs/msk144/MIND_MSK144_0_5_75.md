# MSK144 MIND integration — 0.5.75

MIND is enabled for MSK144 as a candidate ranker, following the same user-facing workflow already used for FT8/FT4.

## Modes

- **Off**: MSK144 decoder runs without MIND.
- **Training**: MIND scores candidates and collects positive/negative examples, but it does not change candidate order.
- **Assist**: after the normal readiness gate, MIND can promote high-confidence time/DF chunks so the classical MSK144 decoder tries them first.

## Safety rule

MIND never validates an MSK144 decode. A message is displayed only after the normal MSK144 path accepts it through synchronization, demodulation, FEC/CRC/unpack and parser checks.

## Feature vector

Each MSK144 candidate chunk is converted to the historical MIND 58×8 input geometry: 464 normalized envelope/edge/energy features around a candidate 144-bit MSK144 frame. This keeps the UI and low-priority trainer shared with FT4/FT8 while allowing MSK144-specific training samples.

## Training labels

Positive samples come from candidate chunks that produce a valid decode. A throttled subset of failed candidate chunks is exported as negative samples to keep the replay buffer balanced without flooding the trainer during wide DF searches.

## Intended use

Start in **Training** during on-air MSK144 monitoring. After enough positive/negative samples have accumulated and the ranker becomes stable, try **Assist** and compare decode responsiveness and CPU behavior.
